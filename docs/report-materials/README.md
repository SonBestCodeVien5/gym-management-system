# Report Materials

This folder stores source material for the formal project report. It is separate from chat handoff
context and from implementation-facing docs.

## Use With

- Use [../system_analysis_design_guide.md](../system_analysis_design_guide.md) to assemble a report structure.
- Use [../faq_why.md](../faq_why.md) for architecture rationale.
- Use [../api_contract.md](../api_contract.md) and current code before stating that an API or rule is implemented.

## Maintenance Rules

- Keep report drafts here even when they are not yet aligned with code.
- Mark planned behavior as planned when it is not in the current backend.
- Summarize durable report decisions into the report files, not into `CHAT_CONTEXT/`.
