# Frontend Delivery Memory

This folder keeps frontend feature memory for React/Vite work.

| Phase | Skill |
|---|---|
| Plan | `$gym-fe-plan` |
| Implement | `$gym-fe-implement` |
| Review | `$gym-fe-review` |
| Test | `$gym-fe-test` |
| Complete | `$gym-fe-complete` |

## Keep

| Path | Purpose |
|---|---|
| `plans/` | Frontend feature plans, pages, state, API contracts, UX decisions |
| `implementations/` | Files changed, decisions, commands, limitations, review handoff |
| `reviews/` | Review findings, fixes, remaining risks |
| `tests/` | Build/manual browser/API verification notes |
| `worklog.md` | Short frontend chronology and roadmap |

## Update Rules

- Keep FE notes concise: screens, routes, components, state/API contracts, commands, risks.
- Update backend API docs only when FE work exposes a backend contract drift that must be fixed.
- Keep frontend build artifacts and dependencies out of tracked memory.
